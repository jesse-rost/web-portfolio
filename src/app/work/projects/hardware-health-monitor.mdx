# Embedded Hardware Health Monitor

This project was a bare-metal embedded systems build for an STM32F411: I used direct register access to collect voltage, current, and temperature data over I2C, drove an LED matrix display, and built a lightweight scheduler to manage sensor polling and UI updates in real time.

Repository: [View on GitHub](https://github.com/jesse-rost/my-portfolio/tree/main/hardware-health-monitor)

## What I built

- A custom scheduler for cooperative multitasking on Cortex-M
- Bare-metal drivers for I2C, ADC, GPIO, and interrupts
- A state-driven LED matrix interface for live health telemetry
- Sensor integration for temperature and current/voltage measurement

## Why it mattered

The project strengthened my embedded systems fundamentals by forcing me to work close to the hardware without HAL abstractions. It was a good exercise in timing, interrupt handling, and designing reliable firmware under tight resource constraints.
