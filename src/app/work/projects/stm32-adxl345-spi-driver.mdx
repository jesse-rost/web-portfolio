# STM32 Bare-Metal SPI Driver

This project focused on low-level embedded driver design for the STM32F411. I built a register-level SPI master driver and a layered ADXL345 interface to read sensor data directly from hardware, without using the STM32 HAL.

Repository: [View on GitHub](https://github.com/jesse-rost/my-portfolio/tree/main/multiprocessing-mandelbrot)

## What I built

- A bare-metal SPI driver for STM32 peripheral registers
- A device-specific ADXL345 driver for configuration and burst reads
- A simple application layer for reading and reporting acceleration data
- Basic verification using hardware timing analysis and loopback testing

## Why it mattered

The project helped me get comfortable with how embedded peripherals really work at the register level. It reinforced the importance of timing, protocol correctness, and hardware-aware software design in embedded systems.
