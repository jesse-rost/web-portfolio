# Concurrent TCP Banking Server

This project was a systems programming exercise in concurrent server design: I built a multithreaded TCP banking server in C that handled multiple clients at once while protecting shared account data with semaphores, mutexes, and a bounded producer-consumer queue.

Repository: [View on GitHub](https://github.com/jesse-rost/my-portfolio/tree/main/tcp-banking-system)

## What I built

- A multiclient TCP server for deposits, withdrawals, and balance checks
- A producer-consumer transaction pipeline to separate request handling from account updates
- Thread-safe synchronization for shared queue and account data
- Persistent balance and transaction logging for basic durability

## Why it mattered

The project was a strong introduction to real-world concurrency and systems design. It highlighted how careful synchronization and queue design are necessary when multiple clients interact with shared state at the same time.
